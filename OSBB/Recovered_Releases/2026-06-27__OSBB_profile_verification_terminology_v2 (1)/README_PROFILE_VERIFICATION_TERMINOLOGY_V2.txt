OSBB — уточнение терминов и готовности профиля (V2)
=====================================================

Зачем это изменение
-------------------
Фраза «данные правильные» была неоднозначной: она смешивала два разных
состояния:

1. обязательные данные заполнены;
2. житель подтвердил показанные обязательные данные.

Теперь эти состояния разделены.

Главное правило
---------------
Пустой parking_time — НЕ правильные и НЕ завершённые данные.

Он означает:

    ⛔ режим парковки не определён
    ⛔ обязательные данные не завершены
    ⛔ подключение телефонного доступа невозможно

Для каждого автомобиля статус парковки должен быть явно определён одним из
трёх значений:

    ☀️ Day
    🌙 Night
    🚫 Не користується паркуванням

Третье значение означает: автомобиль существует, но парковочным местом/тарифом
не пользуется. Это не то же самое, что:

    🚫 Автомобіля немає

Разделение статусов в карточке
-------------------------------
1. ⛔ Обов’язкові дані не завершені
   Есть пустой номер, parking_time, не определён статус «автомобиля нет»
   или есть ожидающая операторская заявка. Телефонный доступ заблокирован.

2. 🟡 Обов’язкові дані заповнені. Потрібне підтвердження мешканця
   Поля уже определены, но житель ещё не подтвердил их.

3. ✅ Обов’язкові дані підтверджені
   Телефонный доступ можно подключать.

4. ℹ️ Додаткові дані для уточнення — не блокують підключення
   Марка/модель и цвет отображаются здесь. Их отсутствие не блокирует доступ
   по текущей политике.

Кнопка
------
Старая неоднозначная кнопка заменяется на:

    ✅ Підтвердити обов’язкові дані

Она появляется только когда все обязательные поля уже заполнены. Она никогда
не показывается при пустом parking_time.

Номер доступа
-------------
Номер, подключаемый к шлагбауму, остаётся отдельным непубличным реквизитом.

Он:
- может отличаться от телефона Telegram;
- может отсутствовать среди контактных телефонов в базе;
- не ищется и не сверяется с ними;
- не изменяет контакты владельца квартиры.

Что меняется технически
-----------------------
Только три Python-файла. SQLite-база и любые существующие данные не меняются.

Установка
---------
Не устанавливайте предыдущий архив с «исправлением кнопки подтверждения»,
если ещё не сделали этого: этот пакет включает правильное исправление и
корректную терминологию.

1. Остановите sandbox bot: Ctrl+C.
2. Распакуйте архив в:

   G:\Programming\Py\OSBB\

3. Запустите:

   RUN_INSTALL_profile_verification_terminology_v2.bat

4. Запустите:

   RUN_CHECK_profile_verification_terminology_v2.bat

   Ожидаемый конец:

   CHECK COMPLETED

5. Запустите:

   Start_OSBB_Live_Services_Sandbox_Bot_v1.bat

Проверка на квартире 174
------------------------
На текущей карточке:
- parking_time = Day, поэтому обязательный режим парковки определён;
- цвет не указан, поэтому это только дополнительное предупреждение;
- после установки появится:

    ✅ Підтвердити обов’язкові дані

После нажатия статус станет готовым для телефонного доступа.

Отдельный будущий тест
----------------------
Для автомобиля с пустым parking_time нужно проверить:
- кнопки Day / Night / Не користується паркуванням;
- создание заявки оператору;
- блокировку телефонного доступа до решения оператора;
- повторную проверку и подтверждение после решения.
