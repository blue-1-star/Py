OSBB — исправление телефонного доступа
====================================

Что исправлено
-------------
1. Клиентский путь «Відкриття телефоном» показывает только один
   понятный пункт «📱 Підключити телефон».
   Внутренний/дублирующий TEST-пункт больше не выводится.

2. До создания намерения житель вводит номер телефона.
   Номер сохраняется вместе с намерением и автоматически переносится
   в настоящий заказ после подтверждения оплаты.

3. Клиентская карточка телефонного доступа показывает номер и цену,
   а не ошибочный текст о количестве пультов.

4. Админ → «📞 Телефонный доступ» открывает отдельную рабочую зону
   только с оплачиваемыми заявками ACCESS. Там нет поставок или
   потребности поставщику.

5. Оператор видит номер в карточке и нажимает один раз
   «📱 Активировать телефонный доступ». Для нового сценария номер
   больше не вводится повторно оператором.
   Старые заказы без сохранённого номера продолжают поддерживаться:
   для них оператор всё ещё может ввести номер вручную.

Установка
---------
1. Остановите live-services sandbox bot: Ctrl+C.
2. Распакуйте этот архив прямо в:
     G:\Programming\Py\OSBB\
   Подтвердите замену:
     Bots\handlers\service_orders_workspace.py
3. Запустите:
     Start_OSBB_Live_Services_Sandbox_Bot_v1.bat

Миграция базы не требуется. Новый пульт и уже завершённые тесты не
изменяются. Это только исправление пользовательского интерфейса и
передачи номера телефона в существующий workflow.

Новый sandbox-тест
------------------
Клиентский режим → «Відкриття телефоном» → «Підключити телефон»
→ введите тестовый номер +380500001174
→ «Зафіксувати намір» → «Передати готівку в O».

Кассир подтверждает наличные обычным путём. Затем:
Админ-режим → «Телефонный доступ» → «Оплачені заявки»
→ открывает заказ → «Активувати телефонний доступ».

После нажатия заказ должен стать «✅ Завершено».
