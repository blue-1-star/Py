OSBB — исправление critical_codes в проверке профиля
====================================================

Причина
-------
При открытии «📋 Перевірити мої дані» бот выдавал:

    NameError: name 'critical_codes' is not defined

В новой логике определялись только `structural_codes`, но ниже интерфейс
правильно пытался использовать `critical_codes`, чтобы показывать кнопки:

- подтвердить отсутствие автомобиля;
- уточнить режим парковки.

Что исправляется
----------------
Теперь определяются оба набора:

1. `critical_codes` — все критичные пункты, для выбора нужных кнопок.
2. `structural_codes` — только незавершённые обязательные данные; именно
   они блокируют кнопку «✅ Підтвердити обов’язкові дані».

Таким образом:
- пустой parking_time остаётся блокировкой;
- когда обязательные данные заполнены, появляется подтверждение;
- кнопки уточнения появляются для соответствующей проблемы;
- марка/модель/цвет остаются дополнительными предупреждениями.

Что не меняется
---------------
- SQLite-база;
- автомобили и parking_time;
- платежи, заказы, доступы;
- контакты и отдельные номера доступа.

Установка
---------
1. Остановите sandbox bot: Ctrl+C.
2. Распакуйте архив в:

   G:\Programming\Py\OSBB\

3. Запустите:

   RUN_INSTALL_profile_critical_codes_fix.bat

4. Запустите:

   RUN_CHECK_profile_critical_codes_fix.bat

Ожидаемый итог:

   CHECK COMPLETED

5. Запустите:

   Start_OSBB_Live_Services_Sandbox_Bot_v1.bat

6. Откройте:

   📋 Перевірити мої дані

Для квартиры 174 должна открыться карточка, а не появиться traceback.
