# ADR-2026-07-13: Создание слоя core_new

**Дата:** 2026-07-13

**Статус:** Принято

**Автор:** San Tretiak

## Контекст

Существующий код OSBB представляет собой монолит, где бизнес-логика
смешана с интерфейсом Telegram-бота. Это усложняет развитие и
поддержку проекта.

Новые функции требуют изменений в коде бота, что создает риск
сломать работающий функционал.

## Решение

Создать новый слой `core_new/`, который будет содержать:

1. **Адаптеры** (`adapters/`) — обертки над старым кодом
2. **Доменную модель** (`domain/`) — чистую бизнес-логику

### Структура


core_new/
├── init.py
├── adapters/
│ ├── init.py
│ └── db_adapter.py # Обертка над Bots/db_access.py
└── domain/
├── init.py
├── vehicles.py # Модель автомобиля
├── residents.py # Модель жителя
├── apartments.py # Модель квартиры
└── payments.py # Модель платежа



### Принцип работы

Старый код **не трогается** и продолжает работать.
Новый код использует **адаптеры** для доступа к старым данным.

## Альтернативы

1. Полный рефакторинг — риск сломать работающий код
2. Оставить как есть — дальнейшее развитие невозможно
3. Создать отдельный микросервис — избыточно для текущих задач

## Последствия

- Постепенная замена старого кода на новый
- Возможность добавлять новые функции без риска
- Четкое разделение ответственности
- Тестируемость новой логики
