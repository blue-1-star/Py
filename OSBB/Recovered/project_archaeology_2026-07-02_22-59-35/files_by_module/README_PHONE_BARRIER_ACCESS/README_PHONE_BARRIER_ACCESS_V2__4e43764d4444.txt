OSBB — телефонный доступ к двум шлагбаумам: рабочая sandbox-модель V2
============================================================================

Назначение
----------
Этот пакет превращает уже созданную схему в рабочий sandbox-сценарий:

Клиент:
  телефон → выбор Дальнего №1 / Ближнего №2 / обоих
  → намерение → подтверждённая оплата

Система:
  → реальный сервисный заказ
  → подписка на телефонный доступ
  → отдельное право для каждого выбранного шлагбаума
  → две строки разового подключения по 200 грн при выборе обоих

Оператор:
  → учётная активация всех выбранных шлагбаумов в sandbox
  → завершение сервисного заказа только когда активированы все выбранные точки.

Реальные контроллеры НЕ подключаются и реальные команды НЕ отправляются.

Что добавляется в этой версии
-----------------------------
1. Нормализованные телефонные запросы до оплаты:
   phone_access_requests
   phone_access_request_points

2. Выбор:
   - Дальний шлагбаум №1;
   - Ближний шлагбаум №2;
   - оба шлагбаума.

3. Расчёт из действующих версионных тарифов:
   - 200 грн подключение за каждый шлагбаум;
   - 100 грн / месяц за каждый шлагбаум;
   - оба: 400 грн подключение, 200 грн / месяц.

4. После подтверждения оплаты:
   - создаётся subscription;
   - создаются отдельные права по каждому шлагбауму;
   - разовое подключение фиксируется как две оплаченные строки по 200 грн;
   - сохраняются тарифные снимки.

5. При активации:
   - создаётся отдельная учётная запись права на каждый шлагбаум;
   - статус каждой точки становится ACTIVE;
   - сервисный заказ получает DIGITAL_ACCESS_ACTIVATED и завершается;
   - реальные команды шлагбаумам НЕ отправляются.

6. Первая абонплата:
   - дата регистрации не позже половины календарного месяца:
     первый период = текущий месяц;
   - позднее половины:
     первый период = следующий месяц.
   - ежемесячные начисления открываются после учётной активации и только для
     активных прав доступа.

Что пока НЕ включено
--------------------
- автоматическое создание платёжных уведомлений по ежемесячной абонплате;
- сверка с долгом за парковку;
- предупреждение и 10-дневная деактивация;
- внешняя интеграция с контроллерами.

Они будут следующим отдельным модулем. Текущая версия уже хранит данные,
необходимые для безопасной реализации этих правил.

Установка
---------
Предпосылка уже выполнена:
- V1 schema migration и её read-only проверка завершились успешно.

1. Остановите работающий live-services sandbox bot:
   Ctrl+C в окне Start_OSBB_Live_Services_Sandbox_Bot_v1.bat.

2. Распакуйте этот архив прямо в:

   G:\Programming\Py\OSBB\

   Важно: не создавайте дополнительную вложенную папку OSBB внутри OSBB.

3. Запустите:

   RUN_INSTALL_phone_barrier_access_v2.bat

   Этот шаг:
   - создаёт резервную копию исходных Python-файлов;
   - обновляет только Python-код;
   - не изменяет SQLite-базу;
   - проверяет синтаксис всех заменённых файлов.

4. После успешного CODE INSTALLATION COMPLETED запустите:

   RUN_MIGRATE_phone_barrier_access_operational_sandbox.bat

   Этот шаг:
   - работает только с:
     Data\db\sandbox\osbb_test_live_services_2026-06-26_20-13-26.db
   - создаёт резервную копию SQLite;
   - добавляет две таблицы запросов;
   - не меняет существующие тесты пультов и старый общий тест телефона.

5. Запустите:

   RUN_CHECK_phone_barrier_access_operational_sandbox.bat

Ожидаемый ответ проверки
------------------------
- operational tables and migration marker present;
- Requests: 0;
- Subscriptions: 0;
- Connection charges: 0;
- Monthly charges: 0.

После проверки запустите обычный live-services sandbox bot:

   Start_OSBB_Live_Services_Sandbox_Bot_v1.bat

Первый обязательный тест после установки
----------------------------------------
Клиентский путь:

1. Клиентский режим.
2. 📞 Открытие по телефону.
3. 📱 Подключить телефон.
4. Выбрать «1️⃣+2️⃣ Обидва шлагбауми».
5. Ввести тестовый номер, например +380501234568.
6. Проверить карточку:
   - оба шлагбаума;
   - подключение 400 грн;
   - абонплата 200 грн / месяц;
   - первая абонплата рассчитана по правилу половины текущего месяца;
   - указано, что проверка парковочного долга требует подтверждения оператора.
7. Зафиксировать намерение.
8. Передать наличные в O и подтвердить их через кассу.
9. Открыть:
   Админ-режим → 📞 Телефонный доступ → 📋 Оплаченные заявки.
10. В карточке должны быть оба шлагбаума и новая подписка в состоянии
    PENDING_ACTIVATION.
11. Нажать «📱 Активувати телефонний доступ».
12. Проверить:
    - обе точки ACTIVE;
    - заказ ✅ Завершено;
    - две оплаченные строки подключения по 200 грн;
    - если первый расчётный период уже наступил, появятся открытые ежемесячные
      строки по 100 грн на каждый активный шлагбаум.

Если любой экран отличается — не продолжайте дальше по сценарию. Пришлите
скриншот и точный текст из окна запуска.
