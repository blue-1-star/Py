# OSBB Assistant — принципы

Версия документа: 0.1

1. По умолчанию Assistant только читает и анализирует состояние проекта.
2. Любое изменение проекта выполняется только по явной команде пользователя.
3. Перед изменением Assistant должен показать план действий.
4. После изменения Assistant должен сформировать отчёт.
5. Любое изменение должно быть обратимым.
6. Assistant не должен предполагать состояние проекта, если может проверить его.
7. Каждая команда должна сообщать корень проекта и текущий Git HEAD.
8. Диагностические команды не должны изменять файлы проекта, Git index или рабочее дерево.

## Версия 0.3

Доступны только безопасные диагностические команды:

- `check`
- `doctor`
- `report`
- `hash`
- `compare`
- `help`
